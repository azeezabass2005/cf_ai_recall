# Ranti Worker

TypeScript backend for Ranti, running on Cloudflare Workers with the [Cloudflare Agents SDK](https://github.com/cloudflare/agents). See the top-level [`../README.md`](../README.md) for the bigger picture and [`../SPEC.md`](../SPEC.md) §1 + §14 for the architecture.

## Layout

```
worker/
├── src/
│   ├── index.ts           ← Hono router (entry point)
│   ├── agent.ts           ← RantiAgent — Durable Object, one per device
│   ├── tools/             ← Tool definitions for the LLM (stubs)
│   ├── routes/            ← REST handlers for the manual form path (stubs)
│   ├── lib/               ← types, auth, llm, errors
│   └── db/migrations/     ← D1 schema
├── wrangler.toml
├── package.json
└── tsconfig.json
```

## Quick start

```sh
npm install
npx wrangler d1 migrations apply ranti-db --local
npm run dev
```

Worker comes up on `http://localhost:8787` (or whatever port you pass with `--port`). Smoke test:

```sh
curl http://localhost:8787/health
curl -X POST http://localhost:8787/chat \
  -H "Content-Type: application/json" \
  -H "X-Ranti-Device: dev-device-123" \
  -d '{"message":"hi"}'
```

## Endpoints

| Method | Path                       | Auth | Status   | Notes |
|--------|----------------------------|------|----------|-------|
| GET    | `/health`                  | no   | ✅ live  | Liveness probe |
| GET    | `/`                        | no   | ✅ live  | Service descriptor |
| POST   | `/chat`                    | yes  | ✅ stub  | Forwards to `RantiAgent` DO; currently echoes |
| GET    | `/reminders`               | yes  | ✅ stub  | Returns `[]` until milestone §11 |
| POST   | `/reminders`               | yes  | 501      | Manual form path — milestone §11 |
| PATCH  | `/reminders/:id`           | yes  | 501      | milestone §11 |
| DELETE | `/reminders/:id`           | yes  | 501      | milestone §11 |
| POST   | `/reminders/:id/pause`     | yes  | 501      | milestone §11 |
| POST   | `/reminders/:id/resume`    | yes  | 501      | milestone §11 |
| GET    | `/nicknames`               | yes  | ✅ stub  | Returns `[]` until milestone §10 |
| POST   | `/nicknames`               | yes  | 501      | milestone §10 |
| DELETE | `/nicknames/:id`           | yes  | 501      | milestone §10 |

Auth is the `X-Ranti-Device` header — anonymous per-install UUID generated by the Android client.

## Tools (LLM-callable)

Defined in `src/tools/`. All are scaffolded with proper Anthropic-style schemas; handlers throw `not implemented` until their milestones land.

| Tool | Milestone |
|------|-----------|
| `create_reminder`, `update_reminder`, `delete_reminder`, `list_reminders`, `pause_reminder`, `resume_reminder` | §7, §11 |
| `resolve_place` | §8 |
| `save_nickname`, `get_nicknames` | §10 |

## Deploy

```sh
npx wrangler login
npx wrangler d1 create ranti-db    # paste database_id into wrangler.toml
npm run db:migrate:remote
npx wrangler secret put ANTHROPIC_API_KEY
npx wrangler secret put GOOGLE_PLACES_API_KEY
npm run deploy
```
